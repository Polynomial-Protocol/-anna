# Anna — macOS AI Tutor

## What this is
Anna is a SwiftUI macOS assistant (macOS 14+) that watches what apps the user
is using, points at UI elements on screen, and can teach the user how to do
things. It has a self-growing knowledge base that should get smarter with
every session.

## Architecture (current)
- `Domain/AssistantEngine.swift` — actor orchestrator. Routes voice/text
  commands through `IntentRouter` to either `DirectActionExecutor` (AppleScript)
  or the AI backend (`AIAPIService` for API keys, `ClaudeCLIService` for
  local CLI).
- `Infrastructure/AI/AIAPIService.swift` — Anthropic + OpenAI + OpenRouter
  HTTP calls with vision. Uses prompt caching for system prompt + tools.
- `Infrastructure/AI/ClaudeCLIService.swift` — spawns local `claude` CLI.
- `Infrastructure/ScreenCapture/ScreenCaptureService.swift` — ScreenCaptureKit,
  max 1280px, JPEG 0.8. Supports full-screen or focused-window capture.
- `Infrastructure/Knowledge/KnowledgeStore.swift` — SQLite + FTS5 + NL
  embeddings, hybrid search.
- `Infrastructure/Knowledge/WikiKnowledgeBase.swift` — Karpathy-style
  `raw/` + `wiki/` + `schema.md` folder structure under
  `~/Library/Application Support/Anna/`. Session logs append-only.
- `Core/AnnaKnowledgeBase.swift` — static self-knowledge (what Anna is).
- `App/AnnaApp.swift` — app entry, menu bar item, hotkey monitors.

## Key constraints
- Prompt caching is enabled on Anthropic (`anthropic-beta:
  prompt-caching-2024-07-31`). System prompt and tools are cached — do not
  mutate them per call or the cache key changes.
- Tutor observations are debounced to a 3s minimum interval in
  `AssistantEngine.executeTutorObservation()`. Do not remove this.
- Context building (`buildHistoryContext`/`buildKnowledgeContext`/
  `buildTourGuideContext`) runs in parallel via `async let`. Keep it that way.
- API keys live in `~/Library/Application Support/Anna/apikey-*` (file-based).
  Keychain migration is silent — never prompt.
- All wiki files are plain markdown in `~/Library/Application Support/Anna/`.
  `raw/` is append-only.
- `schema.md` is generated by the system on first run, not hand-edited.

## Build workflow
Always regenerate the project before building:
```
xcodegen generate && xcodebuild -scheme Anna -configuration Debug -destination 'platform=macOS' build
```
Launch from DerivedData:
```
open ~/Library/Developer/Xcode/DerivedData/Anna-*/Build/Products/Debug/Anna.app
```

## Files to never touch
- `~/Library/Application Support/Anna/schema.md` (system-generated)
- `~/Library/Application Support/Anna/raw/` (append-only)
